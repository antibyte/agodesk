# Internationalization (i18n)

agodesk uses a lightweight JSON-based i18n system under `src/lib/i18n/`.

## Supported UI languages

| Code | Language |
|------|----------|
| `de` | Deutsch (source) |
| `en` | English |
| `fr`, `es`, `zh`, `ja`, `nl`, `pt`, `pl`, `cs`, `it`, `sv`, `no`, `da`, `el`, `hi` | Fully translated |

Setting `system` follows the OS/browser language (`navigator.languages`).

## Adding or updating strings

1. Add the key and German text to [`src/lib/i18n/messages/de.json`](../src/lib/i18n/messages/de.json).
2. Add the English translation to [`src/lib/i18n/messages/en.json`](../src/lib/i18n/messages/en.json).
3. Use the key in Svelte: `{$i18n('your.key')}` or with params: `{$i18n('key', { name: 'value' })}`.
4. In TypeScript services: `getTranslateFn()('your.key')`.

Keys use dot notation (`settings.title`, `chatView.error.connectFailed`).

## Adding a new language

1. Create `src/lib/i18n/messages/xx.json` with translated keys (or start empty for EN fallback).
2. Register the locale in [`src/lib/i18n/locales.ts`](../src/lib/i18n/locales.ts): `APP_LOCALES`, `LOCALE_LABELS`, `LOCALE_BCP47`.
3. Import the JSON in [`src/lib/i18n/loader.ts`](../src/lib/i18n/loader.ts).

## Fallback chain

`locale → en → de` — missing keys in a locale file inherit from English, then German.

## Speech vs UI language

- **UI language** (`AppSettings.locale`): menus, labels, system messages.
- **Gemini speech language** (`AppSettings.speech.language`): BCP-47 tag for live transcription (e.g. `de-DE`). These are independent.
